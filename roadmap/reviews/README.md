# Reviews — 外部レビュー（ChatGPT 等）共有の固定置き場

このディレクトリは、**push 前後のコードレビューを外部レビュアー（ChatGPT 等）に
依頼するための共有ドキュメント**を置く、両者から見える共通の場所。

## なぜ存在するか

これまでレビューは「ローカルで `git diff` をファイルに出し、その都度 ChatGPT に
渡す」という運用だった。それを **GitHub 上の安定した場所**に移し、
- 人間（オーナー）は中継するだけでよくする、
- レビュアー（ChatGPT）は GitHub の固定 URL から自己完結で読めるようにする、
- アシスタント（Claude）は重要点・未解決点だけを報告する、
ための置き場。

## 命名規約

```
roadmap/reviews/<YYYY-MM-DD>-<topic>.md         レビュー依頼パケット（1作業=1ファイル）
```

## 各パケットに含めるもの（自己完結であること）

1. **目的とレビュー観点** — レビュアーが GitHub にアクセスできなくても判断できるよう、
   このファイル単体で完結させる。
2. **コンテキスト** — 何のプロジェクトの何の作業か（最小限）。
3. **変更インベントリ** — 追加/変更ファイルと役割。
4. **設計判断と根拠**。
5. **重点的に見てほしい点（具体的な質問）**。
6. **インライン差分** — コード/文書の差分を貼る。巨大なデータ列（`results/runs/*.jsonl`,
   ラベル `*.jsonl` 等）は**参照のみ**（レビューには不要、可読性のため除外）。
7. **再現/検証手順**。

## 公式開発プロセス（2026-06-13 採用 / D11）

Semantic Resilience Project の標準レビューフローは **PR ベース**:

```
Claude 実装 → ブランチ push → PR 作成
        ↓ （オーナーが PR URL を ChatGPT に渡す）
ChatGPT レビュー
        ↓ （オーナーが ChatGPT のレビューを PR コメントに貼る）
Claude が PR コメントを自動取得 → 修正を push → merge
```

利点:
- **往路**は URL を1つ渡すだけ（public リポジトリなので ChatGPT が直接読める）。
- **復路**は PR コメントに1回貼れば、Claude が `subscribe_pr_activity` で自動取得し、
  オーナーが Claude に転送する手間が消える。
- PR に「変更 / ChatGPT レビュー / Claude の応答 / 最終 diff」が一括で残り、
  **どのように科学的厳密性を獲得したかの研究ノート**になる。

### 各 PR での手順

1. Claude が作業ブランチに実装＋このフォルダにレビューパケット（自己完結）を置き、
   **PR を作成**して `subscribe_pr_activity` で購読。
2. オーナーが **PR URL** を ChatGPT に渡す。
3. ChatGPT のレビューを、オーナーが **PR コメント**に貼る（必要なら該当パケットの
   「## レビュー往復」にも残す）。
4. Claude が PR コメントを取得 → 反映を push → CI 緑 & レビュー解決で **merge**。
   重要点・未解決点のみ報告。

> パケット（`<日付>-<topic>.md`）は引き続き「ChatGPT が単体で読める自己完結の
> レビュー材料」として有効。PR 本文から該当パケットへリンクする。
